OBJECTIVE
---------
Reduce CLAUDE.md from ~2,479 lines to a 600–900 line operating manual by
routing existing content into the new document architecture. Preserve ALL
rationale — nothing that explains a non-obvious "why" is lost; it moves to
DECISIONS.md. This is governed by the CLAUDE.md Maintenance Protocol.

Document architecture (per protocol):
  CLAUDE.md     = current rules, constraints, active state. Terse. Auto-loaded.
  DECISIONS.md  = the "why" (ADR-lite). Repo root. Read before methodology/arch changes.
  Docs/ archive = full superseded specs (v1.6–v1.9 math, retired build sequences).
  git log       = the "what / when". No commit hashes in CLAUDE.md.

================================================================
THE ROUTING RULE (apply to every entry)
================================================================
Ask: "Will an agent take a wrong action without this line?"

  YES, reason obvious      -> keep-as-rule   (terse imperative in CLAUDE.md)
  YES, reason non-obvious  -> split-to-ADR   (terse rule in CLAUDE.md + ADR in DECISIONS.md)
  NO                       -> delete         (the "what" lives in git log)
  Full superseded detail   -> archive-to-Docs (one-line "superseded by, see Docs/" stays)

Regression-guard test: if dropping the narrative would let someone reintroduce a
bug or re-litigate a settled trade-off, the "why" is non-obvious -> split-to-ADR.
When in doubt, default to split-to-ADR. NEVER hard-delete rationale.

================================================================

3. Confirm Docs/ archive location exists for superseded spec detail.

================================================================
PHASE M2 — EXECUTE  (only after Shannon approves TRIAGE.md)
================================================================
Work SECTION BY SECTION. One section = one commit. Never one giant rewrite.

Per approved bucket:
  keep-as-rule    -> rewrite as a single imperative line; append to the numbered
                     rules list (extend the existing 53–74 style; do not prepend prose).
  split-to-ADR    -> write the ADR in DECISIONS.md (newest at top); leave the terse
                     rule in CLAUDE.md with a "see ADR-NNN" link.
  delete          -> remove the block. (Verify the "what" is captured in a commit msg.)
  archive-to-Docs -> move full detail to Docs/<file>; leave a one-line
                     "superseded by vX — see Docs/<file>" reference in CLAUDE.md.

Commit-hash list (item b): delete entirely. git log reproduces it on demand.
Completed phases (item c): collapse each to one "Phase N — COMPLETE" line; move any
  remaining reference-value detail to Docs/.

After each section: commit with a clear message
  e.g. "refactor(docs): migrate Known Fixes vol-source entries -> rules + ADR-012..014"
