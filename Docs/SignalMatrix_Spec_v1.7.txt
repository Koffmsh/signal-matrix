1. Project Overview
Signal Matrix is a multi-timeframe, probabilistic trading signal platform designed to identify high-conviction trade opportunities across a diversified universe of ~51 assets. The platform is built around the principle that simplicity and structural clarity outperform complex over-engineered systems. The methodology is rooted in fractal market theory, wave structure analysis, and probabilistic statistics rather than traditional lagging indicators.

Core Philosophy
Fewer, higher quality signals beat more, lower quality signals
Trend alignment across timeframes is the primary filter
Risk ranges are mathematically derived, not discretionary
The system tells you where the market is, not where you want it to be



2. Methodology

2.1 Timeframe Definitions

Trade — entry/exit timing (≤ 3 weeks)
  Bar window: 5 bars each side to confirm pivot
  H lookback: 63 trading days
  Risk level: LRR + HRR (Bollinger Band framework — see 2.7)

Trend — directional bias filter (≤ 3 months)
  Bar window: 10 bars each side to confirm pivot
  H lookback: 252 trading days
  Risk level: Trend Level (100MA — single floor or ceiling — see 2.8)
  C staleness: 120 trading days → NO_STRUCTURE

Long Term / Tail — macro structural context (display only, ~3 years)
  Bar window: 90 bars each side to confirm pivot
  H lookback: 756 trading days
  Risk level: Tail Level (200MA — single floor or ceiling — see 2.8)
  C staleness: No cutoff (LT structures are inherently old)
  Display label: "Tail" everywhere in UI; code/DB key stays "lt"

Direction values: Bullish / Bearish / Neutral — never Up / Down


2.2 ABC Wave Structure
Four-point pivot structure (A, B, C, D) applies identically to uptrends and downtrends.

Uptrend Structure
  A = pivot low        (e.g. $100)
  B = pivot high       (e.g. $110) — higher high
  C = higher low       (e.g. $105) — C > A confirms uptrend
  D = running high               — established when price closes above B

Downtrend Structure (mirror)
  A = pivot high       (e.g. $100)
  B = pivot low        (e.g. $90)  — lower low
  C = lower high       (e.g. $95)  — C < A confirms downtrend
  D = running low                — established when price closes below B

Pivot Bar Windows
  Trade:    5 bars before AND after — confirms within ~1 week
  Trend:   10 bars before AND after — confirms within ~2 weeks
  Tail/LT: 90 bars before AND after — confirms within ~4.5 months

CRITICAL: Pivot confirmation requires bar_window bars on BOTH sides.
  Pivot high at index i: prices[i] == max(prices[i - bw : i + bw + 1])
  Pivot low  at index i: prices[i] == min(prices[i - bw : i + bw + 1])
  The most recent bar_window bars can NEVER be confirmed pivots.
  D is always a running value — never a confirmed pivot.

CRITICAL: Today's EOD bar IS included in price history.
  history_closes = closes[closes.index.date <= date.today()]
  The scheduler fetches at 4:00 PM ET after market close — today's bar is a confirmed EOD price.
  Do NOT revert to < date.today() — it delays pivot confirmation by one trading day.

C Update Logic — CRITICAL
C is NOT set once and frozen. C updates dynamically as the trend develops.

  # UPTREND — C walks UP (higher lows)
  new_pivot_low = find_most_recent_confirmed_pivot_low(prices, bar_window)
  if new_pivot_low > current_C:
      current_C = new_pivot_low

  # DOWNTREND — C walks DOWN (lower highs)
  new_pivot_high = find_most_recent_confirmed_pivot_high(prices, bar_window)
  if new_pivot_high < current_C:
      current_C = new_pivot_high

  # Break check always uses CURRENT C — never stale C
  if direction == UPTREND and current_price < current_C:  BREAK_OF_TRADE
  if direction == DOWNTREND and current_price > current_C: BREAK_OF_TRADE

Pivot Staleness Thresholds (_STALE_C_DAYS in pivot_engine.py)
  Trade:  C older than  60 trading days → NO_STRUCTURE
  Trend:  C older than 120 trading days → NO_STRUCTURE
  Tail:   No cutoff

ABC Pivot Search — All A Candidates Tried
For each (C, B) pair, all A candidates are iterated newest-first. The engine stops at the first A satisfying C > A (uptrend) or C < A (downtrend). Do not assume the nearest A before B is correct.

Structure Selection Priority in find_abc_structure()
  1. Only one direction found → use it
  2. Both found, only one intact (price on correct side of C) → use intact
  3. Both intact or both broken → most recent C wins, UNLESS:
     a. Newer structure has never established D → older structure governs
     b. Winner spans a prior BREAK_CONFIRMED of same-direction structure → use other

ABC Transition to Bearish After Uptrend Break
  Bearish A = old bullish D    (highest confirmed point)
  Bearish C = first lower high after D  (lower high, C < A, already exists)
  Bearish B = first confirmed lower low (bar_window × 2 bars minimum after break)
  DOWNTREND_VALID fires as soon as bearish B confirms

EXTENDED State — Structural Definition (v1.7)
EXTENDED is a persistent structural state, distinct from the daily overshoot flag.

  # Uptrend EXTENDED: D has pushed more than one full BC range beyond B
  bc_range = abs(B - C)
  if D > B + bc_range: state = EXTENDED; break_level = B  (shifts from C to B)

  # Downtrend EXTENDED (mirror)
  if D < B - bc_range: state = EXTENDED; break_level = B

  Reversion: when new C forms (current D becomes new B, new C established)
             → state returns to UPTREND_VALID / DOWNTREND_VALID
             → break_level returns to new C

Break Level Summary
  Normal structure:      break_level = C (most recent confirmed higher low / lower high)
  EXTENDED structure:    break_level = B (D has exceeded B by more than one bc_range)


2.3 Structural States

  UPTREND_VALID / DOWNTREND_VALID
    Condition: ABC confirmed, price on correct side of C
    Display: Normal green/red

  EXTENDED
    Condition (structural): D > B + bc_range (uptrend) or D < B - bc_range (downtrend)
    Break level shifts from C to B
    Display: State cell shows EXTENDED; ↑/↓ overshoot flag appears on range cell if daily overshoot also fires

  WARNING
    Condition (IV-driven only, never price-driven): LRR drifted below C (uptrend) or HRR drifted above C (downtrend)
    Display: Affected LRR or HRR cell → amber

  BREAK_OF_TRADE
    Condition: One close through C (trade timeframe)
    Forgiveness: Recovery above C on day 1 restores direction
    Display: Trade Dir → Neutral; LRR/HRR grey

  BREAK_OF_TREND
    Condition: One close through C (trend timeframe)
    Forgiveness: Same 1-day recovery rule
    Display: Trend Dir → Neutral; Trend Level grey

  BREAK_CONFIRMED
    Condition: 2+ consecutive closes on wrong side of C
    Recovery: Price must close above B (not just above C) to restore direction
    Display: Permanently Neutral until B recovery

  NO_STRUCTURE
    Condition: Insufficient pivot history OR C older than staleness threshold
    Display: LRR/HRR grey

States that force Neutral: BREAK_OF_TRADE, BREAK_OF_TREND, BREAK_CONFIRMED, NO_STRUCTURE only.
EXTENDED, WARNING, UPTREND_VALID, DOWNTREND_VALID all allow Bullish/Bearish direction.

Intraday violations irrelevant — engine uses EOD closes only.
Trade and Trend states are independent — Trend break does not auto-flip Trade.


2.4 Signal Components

1. Fractal Dimension (D)
D → 1.0 = trending, D → 1.5 = choppy, D → 2.0 = mean-reverting. Derived from Hurst: D = 2 − H.

2. Hurst Exponent (H) — DFA Method
H > 0.5 = trending, H < 0.5 = mean-reverting, H = 0.5 = random walk.
Calculated at Trade (63d), Trend (252d), and Tail/LT (756d) timeframes independently.
H has NO role in direction. H drives conviction and LRR/HRR width only.

DFA Algorithm:
  1. Convert prices to log returns
  2. Compute cumulative sum (integration)
  3. Scales: log-spaced from 10 bars to window//4, ~20 points
  4. For each scale n: fit linear trend per segment, compute RMS of residuals F(n)
  5. H = slope of log(F(n)) vs log(n) via linear regression
  Returns H in [0, 1]

3. Relative Implied Volatility (Rel IV%)
IV expressed as percentile of its own 252-day history. IV Rank formula: (current_iv - min_252) / (max_252 - min_252) × 100.
PRIMARY role: informational display in popup. NOT in conviction formula. NOT in LRR/HRR formula (v1.7).
Source: Schwab options chain (falls back to Yahoo proxy on token expiry).

4. OBV Volume Signal
Applied as multiplier to conviction score. OBV direction determined by applying the same pivot structure logic as the price engine (bar_window = 9) to the OBV series. Compared against Trade Dir (not Viewpoint) — volume is a short-term signal confirming the trade timeframe move.
  Confirming  = OBV direction matches Trade Dir
  Diverging   = OBV direction opposes Trade Dir
  Neutral     = OBV has no structure or Trade Dir is Neutral

5. VIX Macro Regime
Dashboard header indicator — not a per-ticker signal.
  VIX < 20  → Green  (INVESTABLE)
  VIX 20–29 → Amber  (CHOPPY)
  VIX ≥ 30  → Red    (DANGER)


2.5 Viewpoint States — Three States Only

  Bullish: Trade Dir = Bullish AND Trend Dir = Bullish → Conviction calculated
  Bearish: Trade Dir = Bearish AND Trend Dir = Bearish → Conviction calculated
  Neutral: Any other combination                       → Conviction BLANK

No Diverging state. Three states only. Conviction is BLANK when Viewpoint = Neutral.


2.6 Direction Determination — Pivots Only (H has NO role)

  # Uptrend
  if structural_state in (BREAK_OF_TRADE, BREAK_OF_TREND, BREAK_CONFIRMED, NO_STRUCTURE):
      trade_dir = "Neutral"
  elif direction == "uptrend" and current_price > c:
      trade_dir = "Bullish"
  elif direction == "downtrend" and current_price < c:
      trade_dir = "Bearish"
  else:
      trade_dir = "Neutral"

  # Viewpoint
  if trade_dir == "Bullish" and trend_dir == "Bullish":   viewpoint = "Bullish"
  elif trade_dir == "Bearish" and trend_dir == "Bearish": viewpoint = "Bearish"
  else:                                                   viewpoint = "Neutral"

Direction check uses C only (or B when EXTENDED). LRR has no role in direction.


2.7 Trade Timeframe — LRR/HRR: Bollinger Band Framework (v1.7)

SUPERSEDES: All prior sigma/anchor/bc_range/hf/f_hrr/f_lrr formulas. Do not use v1.6 or earlier formulas.

Inputs
  MA20        = 20-day simple moving average of close prices
  STD20       = std(prices[-20:], ddof=0)   ← standard Bollinger Band price-level std
  H_trend     = Hurst exponent, Trend timeframe (252-day DFA) — single H source; H_trade removed
  pivot_dir   = 'uptrend' | 'downtrend'     ← from ABC pivot engine
  ma20_regime = 'uptrend' | 'downtrend'     ← from 2-consecutive-close rule

k Coefficients — 2, Named by Function
  k_wide  = 2.0                        # standard 2σ BB — target side; never flips
  k_tight = max(0, H_trend - 0.5)      # entry side — tight near MA20; clamped ≥ 0

  k_wide:  fixed at 2.0 (standard Bollinger Band). k coefficient refinement deferred.
  k_tight behavior:
    H < 0.50 (mean-reverting) → k = 0.00 → entry = MA20 exactly
    H = 0.60                  → k = 0.10 → entry ≈ MA20 ± 0.1σ
    H = 0.75 (trending)       → k = 0.25 → entry ≈ MA20 ± 0.25σ

Two Inputs Required
  The formula requires BOTH:
  1. Structural direction — from ABC pivot engine (uptrend / downtrend)
  2. MA20 regime — from 2-consecutive-close rule (price above / below MA20)
  These are independent and CAN disagree. Example: SPX 4/8/2026 — structural downtrend,
  but price bounced above MA20 for 2+ days → HRR expanded from tight to BB upper.

MA20 Price Regime Switch — 2-Consecutive-Close Rule
  regime = "uptrend"  if 2+ consecutive closes ABOVE MA20
  regime = "downtrend" if 2+ consecutive closes BELOW MA20

  Independent of ABC pivot structural direction. Pivots say "what is the structural trend."
  Regime says "where is price vs MA20 right now."
  Forgiveness: 1 close on wrong side of MA20 does not flip the regime. Day 2 flips.
  Stored in price_cache.ma20_regime — written on every price fetch.

LRR/HRR Formulas — Structural Direction × MA20 Regime

  STRUCTURAL UPTREND:
    HRR = MA20 + k_wide  × STD20            # always BB upper — never changes

    if ma20_regime == "uptrend":             # normal — price above MA20
        LRR = MA20 - k_tight × STD20        # tight floor ≈ MA20 — buy dip here
    elif ma20_regime == "downtrend":         # counter-trend dip — price below MA20
        LRR = MA20 - k_wide  × STD20        # wide floor = BB lower — dip expanded

  STRUCTURAL DOWNTREND (mirror):
    LRR = MA20 - k_wide  × STD20            # always BB lower — never changes

    if ma20_regime == "downtrend":           # normal — price below MA20
        HRR = MA20 + k_tight × STD20        # tight ceiling ≈ MA20 — sell bounce here
    elif ma20_regime == "uptrend":           # counter-trend bounce — price above MA20
        HRR = MA20 + k_wide  × STD20        # wide ceiling = BB upper — bounce expanded

Key Behavior
  Target side NEVER moves — always standard 2σ BB. Structural direction determines which side is target.
  Entry side flips on 2 consecutive closes crossing MA20. When price is on the "normal" side
  (above MA20 in uptrend, below in downtrend) → entry is tight near MA20. When price crosses
  to the counter-trend side → entry widens to full BB band.
  SPX 4/8/2026 is a confirmed example: structural downtrend, counter-trend bounce above MA20
  → HRR expanded from tight (≈MA20) to BB upper (MA20 + 2σ).

Self-Correction Property (no hard clamps needed)
  When close drops below LRR → tomorrow's MA20 falls (new low enters window, old high exits)
  → LRR follows MA20 downward automatically. Same in reverse for uptrend HRR breaches.
  No hard floor/ceiling against C or D required — the formula self-heals within 1–3 sessions.

H Source — Single: H_trend (252-day)
  H_trade (63-day) removed entirely — too noisy for k coefficients and conviction scoring.
  H_trend drives k_tight. k_wide is fixed at 2.0 (standard BB). MA20 provides the daily
  dynamic movement; H_trend provides the stable entry-side width.

Display
  LRR/HRR always display regardless of viewpoint.
  Color: Bullish → green (#00e5a0) | Bearish → red (#ff4d6d) | Neutral → grey (#8899aa)
  Each cell uses its OWN timeframe direction for color, not overall viewpoint.

Daily Overshoot Flag (tactical, separate from structural EXTENDED state)
  Reads prior_hrr / prior_lrr from yesterday's signal_output before overwriting.

  Uptrend:   if today_close > prior_hrr → hrr_overshoot = True  (↑ flag, "do not chase" tooltip)
  Downtrend: if today_close < prior_lrr → lrr_overshoot = True  (↓ flag, "do not chase" tooltip)

  This is NOT a structural state. State cell still shows UPTREND_VALID / DOWNTREND_VALID.
  Structural EXTENDED (D > B + bc_range) shows "EXTENDED" in state cell.


2.8 Trend and Tail Timeframes — Single Moving Average Level (v1.7)

SUPERSEDES: Dual LRR/HRR bands for Trend and LT/Tail timeframes. Rel IV scaling removed entirely from Trend/Tail.

Rationale
  LRR adds no value for managing risk on the correct side of a Trend or Tail downtrend.
  Only one level matters per regime: the floor in an uptrend, the ceiling in a downtrend.
  The MA level is both the floor (uptrend) and the ceiling (downtrend).

Trend Level — 100MA
  MA100 = 100-day simple moving average of close prices
  Slope  = MA100[today] - MA100[10 trading days ago]   # + = bullish slope; - = bearish slope

  Uptrend (slope > 0 AND Trend Dir = Bullish):
    Trend Level = MA100   # floor — support on pullback; add to longs here
    Display: green

  Downtrend (slope < 0 AND Trend Dir = Bearish):
    Trend Level = MA100   # ceiling — resistance on bounce; sell/short here
    Display: red

  Neutral OR slope direction contradicts Trend Dir:
    Trend Level = None    # hidden — no level displayed

  Break level: Pivot C (or Pivot B if structural EXTENDED state)
  Display column: "Trend Level" (replaces "Trend LRR")
  "Trend HRR" column: removed from table and popup

Tail Level — 200MA
  MA200 = 200-day simple moving average of close prices
  Slope  = MA200[today] - MA200[20 trading days ago]   # longer slope window for LT

  Uptrend (slope > 0 AND LT Dir = Bullish):
    Tail Level = MA200    # structural floor — macro support
    Display: green

  Downtrend (slope < 0 AND LT Dir = Bearish):
    Tail Level = MA200    # structural ceiling — macro resistance
    Display: red

  Neutral:
    Tail Level = None

  Break level: Pivot C (or Pivot B if structural EXTENDED state)
  Display column: "Tail Level" (replaces "LT LRR")
  Code/DB key: "lt" — unchanged everywhere in code, models, and database
  Display label only: "Tail Dir", "Tail Level", "Tail State" in popup and table header


2.9 Conviction Score — v1.7

Base Score
  Base = H_trend × 100   # H_trade removed — single H source (252-day DFA)

Proximity Boost (direction-aware)
  The proximity score measures how close price is to the entry zone.
  Entry zone = LRR for Bullish, HRR for Bearish.

  Bullish (entry zone = LRR):
    prox = 1 - (close - trade_lrr) / (trade_hrr - trade_lrr)
    prox = 1.0 when close = LRR (at the floor — ideal entry)
    prox = 0.0 when close = HRR (at the target)

  Bearish (entry zone = HRR):
    prox = (close - trade_lrr) / (trade_hrr - trade_lrr)
    prox = 1.0 when close = HRR (at the ceiling — ideal short entry)
    prox = 0.0 when close = LRR (at the profit target)

  Clamp: prox = max(0.0, min(1.0, prox))
  Combined: conviction_raw = Base × (0.70 + 0.30 × prox)

OBV Volume Multiplier (unchanged)
  Confirming → × 1.15   (OBV direction matches Trade Dir)
  Neutral    → × 1.00
  Diverging  → × 0.80   (OBV direction opposes Trade Dir)

Final Conviction = conviction_raw × OBV_multiplier

CRITICAL: Conviction is BLANK when Viewpoint = Neutral.
Stored fields: conviction (final, existing field). h_score and proximity_score may be added for debugging.


2.10 ENTRY Signal — v1.7 (Proximity-Based)

SUPERSEDES: 2%-of-price absolute threshold. Replaced with range-normalized proximity.

BUY Conditions (all four must be true)
  Viewpoint = Bullish
  Trade Dir = Bullish
  Trend Dir = Bullish
  prox_bullish > 0.85
    where: prox_bullish = 1 - (close - tradeLRR) / (tradeHRR - tradeLRR)

SELL Conditions (all four must be true)
  Viewpoint = Bearish
  Trade Dir = Bearish
  Trend Dir = Bearish
  prox_bearish > 0.85
    where: prox_bearish = (close - tradeLRR) / (tradeHRR - tradeLRR)

Rationale: 2%-of-price was not normalized to instrument volatility.
  HRR - LRR is derived from STD20 → already volatility-scaled per instrument.
  prox > 0.85 = within bottom 15% of the range (from entry side) for any ticker.
  Example: SPY range = $40 → ENTRY fires within $6 of LRR.
           NVDA range = $8 → ENTRY fires within $1.20 of LRR. Both correct.

Neutral viewpoint never triggers ENTRY signal regardless of price proximity.
ENTRY count displayed in header summary row alongside BULLISH / BEARISH / ALIGNED / ALERTS.

Display: ▲ BUY (green #00e5a0) | ▼ SELL (red #ff4d6d) | blank when conditions not met.

7. Statistical Framework

  Component                  | Paradigm      | Reason
  Hurst Exponent             | Frequentist   | Objective measurement of price series property
  Fractal Dimension          | Frequentist   | Derived from H: D = 2 − H
  Bollinger Band LRR/HRR     | Frequentist   | MA20 ± k×STD20; k modulated by H
  Rel IV Percentile          | Frequentist   | Rank within own 52-week history
  Conviction Score           | Frequentist   | H-based + proximity to entry zone
  OBV Pivot Direction        | Frequentist   | Structural pivot logic on OBV series
  Trend / Tail Level         | Frequentist   | MA100 / MA200 slope as directional floor/ceiling
  Quad Probability Dist.     | Bayesian      | Continuously updated belief across 4 quads (future)
  Forward Quarter Projections| Bayesian      | Prior decay without confirming evidence (future)
