1. Project Overview
Signal Matrix is a multi-timeframe, probabilistic trading signal platform designed to identify
high-conviction trade opportunities across a diversified universe of ~51 assets. Built around
the principle that simplicity and structural clarity outperform complex over-engineered systems.
Methodology is rooted in fractal market theory, wave structure analysis, and probabilistic
statistics.
Core Philosophy
- Fewer, higher quality signals beat more, lower quality signals
- Trend alignment across timeframes is the primary filter
- Risk ranges are mathematically derived, not discretionary
- The system tells you where the market is, not where you want it to be

2.2 ABC Wave Structure
Four-point pivot structure (A, B, C, D) that applies identically to uptrends and downtrends.
Uptrend Structure
Point    Example Price    Description
A        $100             Pivot low
B        $110             Pivot high -- higher high
C        $105             Higher low -- C > A confirms uptrend
D        $120             Running high -- updates as price climbs above B
Downtrend Structure
Point    Example Price    Description
A        $100             Pivot high
B        $90              Pivot low -- lower low
C        $95              Lower high -- C < A confirms downtrend
D        $85              Running low -- updates as price falls below B
Pivot Bar Windows
Timeframe    Bar Window    Confirmation Rule
Trade        5 bars        5 bars before AND 5 bars after required
Trend        20 bars       20 bars before AND 20 bars after required
Long Term    90 bars       90 bars before AND 90 bars after required
---
C Update Logic -- CRITICAL
C is NOT set once and frozen. C updates dynamically as the trend develops.
# UPTREND -- C walks UP (higher lows)
new_pivot_low = find_most_recent_confirmed_pivot_low(prices, bar_window)
if new_pivot_low > current_C:
    current_C = new_pivot_low  # Update to higher low
# DOWNTREND -- C walks DOWN (lower highs)
new_pivot_high = find_most_recent_confirmed_pivot_high(prices, bar_window)
if new_pivot_high < current_C:
    current_C = new_pivot_high  # Update to lower high
# Break check always uses CURRENT C -- never stale C
if direction == UPTREND and current_price < current_C: BREAK_OF_TRADE
if direction == DOWNTREND and current_price > current_C: BREAK_OF_TRADE
Example -- GLD trade timeframe:
Initial C = $427.13  Feb 2   (first confirmed higher low)
Updated C = $448.20  Feb 17  (new higher low -- C walks up)
Break of trade = price closes below $448.20 (current C, not $427.13)
---
ABC Transition to Bearish -- Dual-Role Pivot (CRITICAL)
When an uptrend breaks, the bearish structure builds using pivots that already exist.
The dual-role pivot is the key insight:
  Bearish A = old bullish D        (the highest confirmed point -- pivot high)
  Bearish C = old bullish new B    (the lower high that formed before the break --
                                    already confirmed with bar_window on both sides,
                                    satisfies C < A immediately)
  Bearish B = first lower low      (confirms AFTER the break of bullish C)
  Bearish D = running low          (tracks down once DOWNTREND_VALID fires)
Sequence after uptrend break:
1. Price closes below bullish C         -> BREAK_OF_TRADE, Trade = Neutral
2. Old bullish new B becomes bearish C  -> already confirmed, C < bearish A
3. Price makes first lower low          -> bearish B confirms (bar_window both sides)
4. DOWNTREND_VALID fires               -> Trade = Bearish
5. Bearish D tracks lower              -> running low
Why DOWNTREND_VALID fires quickly: bearish C was already confirmed before the break.
The engine only needs bearish B to confirm -- then the full A/B/C structure exists.
The same logic applies in reverse for a bearish-to-bullish transition:
  Bullish A = old bearish D        (lowest confirmed point)
  Bullish B = old bearish C        (the lower high that becomes the first higher high > A)
  Bullish C = first higher low     (confirms after the break, C > A)
  Bullish D = price closes above B -> UPTREND_VALID fires
---
Pivot Staleness Thresholds
If pivot C is stale (too old), the structure is unreliable. Override to NO_STRUCTURE:
Timeframe    Max C Age          Rationale
Trade        60 trading days    Trade timeframe -- stale C after ~3 months is meaningless
Trend        100 trading days   Trend timeframe -- stale C after ~5 months unreliable
Long Term    No cutoff          LT structures are inherently old by design
Implementation: use pandas_market_calendars for trading day count (already imported).
If C date > threshold -> override state to NO_STRUCTURE, null all pivot values.
---
Key Rules
- D is established when price closes above B (uptrend) or below B (downtrend)
- D is a running value -- updates continuously, never a confirmed pivot
- Today's incomplete bar must be excluded before pivot detection runs
- Pivot confirmation requires bar_window bars on BOTH sides -- before AND after
- The ABC structure always references the most recent confirmed pattern
- C updates dynamically -- always the most recent confirmed higher low / lower high

2.3 Signal Components
1. Fractal Dimension (D)
- D -> 1.0 = trending, D -> 1.5 = choppy, D -> 2.0 = mean-reverting
- Derived from Hurst Exponent: D = 2 - H
2. Hurst Exponent (H) -- DFA Method
Timeframe    DFA Lookback                  Role
Trade        63 trading days (~3 months)   Primary signal -- drives conviction score
Trend        252 trading days (~1 year)    Alignment filter -- confirms direction
Long Term    756 trading days (~3 years)   Context only -- popup display, not in conviction formula
3. Relative Implied Volatility (Rel IV%)
- IV expressed as percentile of its own 52-week range
- PRIMARY role: scales LRR/HRR width (ATR equivalent)
- Rel IV is NOT in the conviction score formula -- LRR/HRR width scaling only
- High IV = wider range = trade breathes more. Low IV = tighter range = precise entry.
4. OBV (On-Balance Volume) Signal -- REPLACES price-momentum proxy
- Applied as a multiplier to conviction score after base score is calculated
- OBV direction determined by pivot structure applied to OBV series (bar_window = 9)
- Confirming: OBV direction matches viewpoint direction
- Diverging: OBV direction opposes viewpoint direction
- Neutral: OBV direction mixed or no confirmed structure
5. VIX Macro Regime (dashboard header indicator -- Phase 4)
- Not a per-ticker signal -- a market-wide regime read
- VIX < 19   -> Green  -- investable, VCFs mechanically adding equity exposure
- VIX 20-29  -> Amber  -- choppy, reduce size, tighter stops
- VIX >= 30  -> Red    -- danger, defensive posture, minimal exposure
- Reads from existing VIX price_cache -- no new data fetch needed

2.6 Direction Determination -- Pivots Only
H has no role in direction. H drives conviction and LRR position only.
Direction check -- C is the only gate. LRR has no role in direction determination.
# UPTREND direction
if structural_state == "BREAK_OF_TRADE":
    trade_dir = "Neutral"
elif current_price > c:          # price above invalidation level
    trade_dir = "Bullish"
else:
    trade_dir = "Neutral"
# DOWNTREND direction
if structural_state == "BREAK_OF_TRADE":
    trade_dir = "Neutral"
elif current_price < c:          # price below invalidation level
    trade_dir = "Bearish"
else:
    trade_dir = "Neutral"
# Viewpoint
if trade_dir == "Bullish" and trend_dir == "Bullish": viewpoint = "Bullish"
elif trade_dir == "Bearish" and trend_dir == "Bearish": viewpoint = "Bearish"
else: viewpoint = "Neutral"
Direction lookup table:
Condition                                     Direction
Price above C -- uptrend, C unviolated        Bullish
Price below C -- downtrend, C unviolated      Bearish
Break of Trade (price closes through C)       Neutral
FORMING -- no new C confirmed yet             Bullish or Bearish (price vs C still checked)
EXTENDED -- price beyond D, no new C          Bullish or Bearish (price vs C still checked)
WARNING -- IV-driven range expansion          Bullish or Bearish (price vs C still checked)
NO_STRUCTURE -- insufficient pivot history    Neutral
BREAK_OF_TRADE                                Neutral (explicit override)
BREAK_OF_TREND                                Neutral for trend timeframe only
CRITICAL: FORMING, EXTENDED, and WARNING do NOT force Neutral.
Direction is always determined by price vs C -- structural state affects display only.
---
Break of Trade / Recovery Behavior (LOCKED)
One close below C    -> BREAK_OF_TRADE fires immediately, Trade = Neutral
Price recovers above C  -> Trade = Bullish restored (engine recalculates fresh each run)
Intraday violations  -> Irrelevant -- engine uses EOD closes only
Two-close rule       -> Explicitly rejected -- not implemented
Position management at break (trader decision framework, not engine logic):
- Trade breaks, Trend intact  -> Reduce to minimum position, watch for recovery or Trend break
- Trade breaks, Trend breaks  -> Full exit, zero position -- no discretion
- Trade recovers above C      -> Re-engage at LRR if conviction supports

2.8 LRR / HRR Naming Convention
LRR (Lower Risk Range) = always the lower price value.
HRR (Higher Risk Range) = always the higher price value.
Names are directional by price level, not by trend direction.
Trend Direction    Entry Level                           Profit Target
Uptrend            LRR -- enter long at or near level    HRR -- profit target above D
Downtrend          HRR -- enter short on bounce to C     LRR -- profit target below D

2.9 LRR / HRR Calculation
LRR Formula -- Uptrend
Base LRR = C + (B - C) x H_factor
H_factor lookup:
  H > 0.65        ->  0.95  (near B -- strong trend)
  H 0.55 - 0.65   ->  0.50  (midpoint -- moderate trend)
  H 0.50 - 0.55   ->  0.05  (near C -- weak trend)
  H < 0.50        ->  0.00  (defaults to C -- LRR shown grey)
LRR always calculates and always displays.
H < 0.50 defaults LRR to C, shown in grey.
HRR Formula -- Uptrend
Base HRR = D + (1-sigma of recent returns)
HRR vs prior D interpretation:
  HRR well above prior D  -> full conviction, full size
  HRR near prior D        -> moderate conviction, reduced size
  HRR below prior D       -> caution confirmed, minimum size
Downtrend Mirror Formulas
Base HRR = C - (C - B) x H_factor   (entry on bounce -- higher price)
Base LRR = D - (1-sigma of recent returns)  (profit target -- lower price)
Rel IV Scaling (ATR Equivalent)
Rel IV%    LRR Adjustment (Uptrend)    HRR Adjustment (Uptrend)
0-30%      x 0.99 (tight)             x 1.02
31-60%     x 1.00 (neutral)           x 1.05
61-80%     x 0.97 (wider)             x 1.10
> 80%      x 0.94 (maximum)           x 1.15
Downtrend: HRR pushed up, LRR pulled down.
No hard floor or ceiling -- IV can push LRR below C or HRR above C -> WARNING state.

2.11 OBV Pivot Engine (v1.5 -- NEW)
Replaces the price-momentum proxy (_volume_signal) entirely.
Rationale
- Price 5-day / 20-day momentum proxy was redundant with Hurst and provided no independent signal
- OBV (On-Balance Volume) measures net volume flow -- independent from price trend persistence
- OBV structural breakout leads price: OBV makes new highs before price breaks B or D
- During a mature bullish trend, OBV impulses, consolidates, and impulses again
- OBV breaking its prior structural high signals the start of a new price impulse
OBV Calculation
def _build_obv(closes, volumes):
    obv = [0.0]
    for i in range(1, len(closes)):
        if closes[i] > closes[i-1]:
            obv.append(obv[-1] + volumes[i])
        elif closes[i] < closes[i-1]:
            obv.append(obv[-1] - volumes[i])
        else:
            obv.append(obv[-1])
    return obv

- Same pivot logic as price pivot engine -- applied to OBV series
- Bar window: 9 bars (both sides required -- same rule as price pivot engine)
- Bullish OBV: higher highs AND higher lows in OBV series
- Bearish OBV: lower highs AND lower lows in OBV series
- Neutral: mixed or insufficient confirmed pivots

When viewpoint transitions TO Bullish or Bearish (Trade + Trend align):
- Timestamp is recorded in signal_output.viewpoint_since (ISO format, ET timezone)
- Clock resets on any new alignment transition (Neutral -> Bullish, Bullish -> Bearish, etc.)
- Clock preserves existing timestamp when viewpoint stays in same aligned state
- Timestamp clears (null) when viewpoint = Neutral
- Displayed in popup as "Aligned since" -- hidden when Neutral
- Purpose: trade duration analytics, future backtesting of signal persistence

2.14 Multi-Timeframe Signal Logic Flow
1. Long Term H -> structurally trending? (context only -- popup display)
2. Trend H -> trending over 3 months? (alignment filter)
3. Trade H -> short term aligned? (primary signal)
4. ABC pivot structure -> where are we in the wave?
5. OBV pivot direction -> is volume flow confirming the trend?
6. Rel IV -> where in the volatility distribution? (LRR/HRR width only)
7. VIX regime -> what is the macro investability environment?
8. Output -> LRR, HRR, structural state, conviction score (0-100%)
Primary filter: Trade and Trend must be aligned (same direction) for Viewpoint to be Bullish/Bearish.

- Price history excludes today's incomplete bar before pivot detection
- Auto-loading from SQLite cache on page load IS allowed (local DB read, not Yahoo call)
- All cache_date and run_date writes use ET date -- never UTC

8. Statistical Framework
Component                       Framework      Rationale
Hurst Exponent (H)              Frequentist    Objective measurement of price series property
Fractal Dimension (D)           Frequentist    Derived from H: D = 2 - H
Gaussian Return Distribution    Frequentist    Historical return frequency -> confidence intervals
Relative IV Percentile          Frequentist    Rank within own 52-week historical distribution
Conviction Score                Frequentist    Trend persistence measurement (H-based)
LRR / HRR Ranges                Frequentist    Anchored to Gaussian sigma bands and pivot points
OBV Pivot Direction             Frequentist    Structural pivot detection applied to volume flow
Quad Probability Distribution   Bayesian       Continuously updated belief across 4 quads
Forward Quarter Projections     Bayesian       Prior decay without new confirming evidence
Policy Signal Modifiers         Bayesian       Discrete evidence updates to forward projections
